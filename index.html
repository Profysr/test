    Stages or Workflow of github

  • untracked - abbhi tk add ni kya iseee, matlab git ko abbhi is file ka ni pta
  • Modified - wo file jisee pehle add to kya hwa hai lekin usee change kya hai phir se
  • Add - commit karne k lye tayaar kardein hai
  • Commit - record of change,matlab snapshot le leta hai change ka or usee .git mein save karleta hai
ak cheez or hai k sometimes hum github pr pehle repo banante hai then usee locally clone karke us pr work start karte hain jis se jb bhi hum koi change karein ge files mein to commit karne k bd wo humein btaye ga you re ahead by 1 commit from your remote repo, then humein usee push karna padde ga k jo changes hamare local pr hain wo remote pr bhi show hun jiske lye command yh hai

git push -u origin main

jis mein -u ak flag hai jo --setup-upstream ko show kar rhaa hai means ab hum jb bhi dobara push karna padde ga code, to hum sirf likhein ge github push or agr yhi hum pehle code likhein then bd mein khud repo ko upload krein, to uske lye kuch steps hain jo follow karna paddte hain,
  1. git init --- is se ak repo initialize ho jti hai locally
  2. git add . --- then hum files ko add karte hain
  3. git commit -m "message" --- phir files ko commit karte hain jis se wo ak snapshot le kr hamari .git file mei save kardeta hai
  4. git branch -- yh btata hai abbi hum konsi branch hai or total kitni branches hain
  5. git branch -m newName --- branch ko rename kar sakte hain ise
  6. git remote add origin "link" --- is cmd mein git remote add repo -- git pr add karde ga jiska naam origin hoga or next usii repo ka link jo github pr hai wo dena hai
  7. git remote -v --- verify karte hai remote ko
  8. git push -u origin main --- code ko remote pr push karde ga, jiska naam origin hoga
